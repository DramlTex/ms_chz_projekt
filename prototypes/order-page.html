Заказ кодов маркировки

Выберите карточку из Национального каталога, подготовьте список кодов идентификации, подпишите документы УКЭП и отправьте заказ в True API и СУЗ. Страница демонстрирует пользовательский поток с визуализацией процессов.

1. Выбор карточки из НК

Укажите GTIN или ID карточки и ИНН держателя. Доступно автоопределение через deeplink с параметрами `gtin` и `inn`.

14 цифр, автоматически проверяется с карточкой.
Сценарий заказа

API `/v3/product` Проверка выполняется в фоновом режиме, результат фиксируется в карточке ниже.

2. Карточка товара

Автозаполнение полей заказа Данные доступны для редактирования до отправки.

3. Проверка кодов идентификации

Вставьте КИ построчно. Сервис НК допускает до 100 кодов за один запрос. Ошибки возвращаются с расшифровкой.

API `/v3/mark-check` Проверка обязательна до подписания заказа.

4. Дополнительные документы

Добавьте файл загрузочного фида или сопроводительные документы. Блок активируется после выбора товара.

    Необязательный шаг Файлы хранятся локально до подтверждения заказа.

    5. Подписание документов УКЭП

    Выберите сертификат CryptoPro и подпишите пакет документов перед отправкой заказа.

    CryptoPro Browser Plug-in выполняет подпись ГОСТ 34.10-2012.
    API `/auth/key` · CryptoPro Подписанный hash сохраняется в памяти до отправки.

    6. Отправка заказа

    После проверки КИ и подписи запрос автоматически отправится в True API и СУЗ.

    Отправка фиксируется в аналитике (`order.create_attempt`).

    7. Таймлайн статусов

    Прогресс документов отражает ответы True API и СУЗ. Обновляется по WebSocket `/ws/orders/:id`.

      Время синхронизации обновляется по мере получения событий.

      8. Уведомления

      Сообщения воркера отображаются в реальном времени: ошибки подписи, статус закрытия заказа, предложения по выбытиям.

        WS `/ws/orders/:id` События `ORDER_CLOSED`, `ORDER_FAILED` сопровождаются рекомендациями.
